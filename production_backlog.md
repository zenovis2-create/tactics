# Production Backlog v1
## Project: 잿빛의 기억

## 1. 문서 목적

이 문서는 지금까지 정리한 세계관, 캠페인, 전투, 데이터, 플래그, 장비 문서를 실제 구현 순서로 바꾸기 위한 제작 백로그 문서다.

이 문서가 고정하는 것은 아래 여섯 가지다.

1. 어떤 순서로 무엇을 만든다
2. 어떤 시스템이 어떤 챕터보다 먼저 와야 하는가
3. 각 마일스톤의 완료 기준은 무엇인가
4. Codex에 어떤 크기로 작업을 쪼개서 던질 것인가
5. 어디서 구조를 얼리고 어디서 확장할 것인가
6. 일정이 밀릴 때 무엇을 자르고 무엇을 유지할 것인가

이 문서는 “좋은 아이디어 목록”이 아니라 **실제 구현 우선순위 문서**다.

---

## 2. 제작 원칙

### 2-1. 항상 플레이 가능한 빌드를 유지한다
어떤 시점에도 프로젝트는 아래 둘 중 하나는 만족해야 한다.

- 최소 한 개의 전투 스테이지가 시작부터 끝까지 플레이 가능
- 최소 한 개의 캠페인 구간이 스테이지 → 컷신 → 캠프까지 이어서 동작

즉, 거대한 미완성 시스템을 동시에 여러 개 깔지 않는다.

### 2-2. 한 번에 하나의 새 메커닉만 연다
새 메커닉은 아래 순서로 연다.

- 전투 기본 루프
- 보물상자 / 악세사리
- 방어구
- 첫 보스 무기
- 랜덤 무기 / 회상 토벌전
- 분해 / 문장 장부
- 대장간
- 문장 조율
- 선택 제작 / 옵션 교정
- 최종장 전용 공명 / 이름 앵커

이 순서를 깨면 서사보다 시스템이 먼저 튀어나와서 템포가 망가진다.

### 2-3. 장 구현보다 시스템 안정화가 먼저다
장 문서가 이미 잘 정리돼 있어도, 아래 시스템이 없으면 다음 장으로 가지 않는다.

현재 `M0 ~ M2` 기준 최소 선행 시스템은 아래다.

- 전투 기본 루프
- 스테이지 로딩
- 이동 / 공격 / 대기 / 반격
- 턴 매니저와 승패 조건
- 최소 컷신/플래그 핸드오프
- 최소 캠프 진입 루프

반면 아래는 **전투 MVP 이후**에 단계적으로 붙여도 된다.

- 세이브/로드
- 장비 적용
- 랜덤 보스 드롭
- 회상 토벌전
- 선택 제작 / 옵션 교정
- 수익화 / IAP / DLC / 스토어 자산

### 2-4. 데이터와 로직을 분리한다
- 맵 배치: Scene
- 규칙: StageData
- 수치: Resource
- 텍스트: JSON
- 진행 결과: SaveData

이 원칙을 어기는 작업은 구현은 빨라 보여도 나중에 전부 무너진다.

### 2-5. 한 작업은 하나의 완료 기준을 가진다
Codex에 던지는 작업은 반드시 아래를 포함해야 한다.

- 읽을 문서
- 구현 범위
- 수정 가능 파일 범위
- 완료 기준
- 테스트 기준
- 남겨도 되는 TODO

## 3. 완료 기준 정의

각 마일스톤은 아래 기준을 모두 만족해야 “완료”다.

### 기술 완료
- 프로젝트가 열리고 스크립트 에러가 없다
- 저장/로드가 필요한 기능은 실제로 저장된다
- 씬 전환이 끊기지 않는다
- 기본 smoke test를 통과한다

### 디자인 완료
- 문서에 정의한 메커닉이 최소 1회 실제 플레이로 검증됐다
- 템포가 다음 마일스톤과 충돌하지 않는다
- 메커닉이 이후 챕터에서도 재사용 가능하다

### 콘텐츠 완료
- 대상 챕터/서브장의 핵심 전투 흐름이 구현됐다
- 컷신/기억/물증/캠프 대화가 최소 연결된다
- 보상과 시스템 해금 타이밍이 문서와 일치한다

## 4. 우선순위 체계

이 문서에서는 우선순위를 아래처럼 쓴다.

- **P0**: 이게 없으면 다음 단계로 못 감
- **P1**: 강하게 필요함, 바로 다음 단계
- **P2**: 있으면 좋지만 다음 마일스톤을 막지는 않음
- **P3**: 후반 품질/편의/확장

마일스톤 안에서도 P0를 먼저 끝내고 P1, P2로 내려간다.

---

## 5. 백로그 구조

백로그는 세 축으로 나눈다.

### A. Foundation / Systems
엔진, 전투, 데이터, 세이브, UI 기반

### B. Campaign Content
챕터, 컷신, 맵, 보상, 동료 합류

### C. Meta / Progression
장비, 파밍, 캠프, 대장간, 문장, 엔딩 분기

각 마일스톤은 이 셋을 함께 움직이되,  
항상 **Foundation → Campaign → Meta** 순으로 잠근다.

## 6. 전체 마일스톤 개요

### M0. 프로젝트 뼈대
- 목표: 프로젝트 구조, 데이터 레지스트리, 세이브/플래그 기반 완성
- 상태: 시작점

- 목표: 기본 전투 1판이 끝까지 돈다
- 챕터 범위: `1-1`, `1-5`

### M2. 챕터 1 완성 + 캠프 기본
- 목표: 1장 전체와 캠프 이동 루프 완성
- 챕터 범위: `1-2 ~ 1-4` + 인터루드

### M3. 챕터 2 + 악세사리/상자 루프
- 목표: 브란 합류, 보물상자, 악세사리 장착 루프
- 챕터 범위: `2-1 ~ 2-5`

### M4. 챕터 3 + 방어구 + 첫 보스 무기
- 목표: 티아 합류, 안개/함정, 방어구, 첫 고정 보스 무기
- 챕터 범위: `3-1 ~ 3-5`

### M5. 챕터 4 + 첫 랜덤 무기 + 회상 토벌전
- 목표: 세린 장, 수위 변화, 랜덤 무기, 첫 hunt
- 챕터 범위: `4-1 ~ 4-5`

### M6. 챕터 5 + 분해 + 문장 장부
- 목표: 에녹 합류, 화재/봉인, salvage, sigil ledger
- 챕터 범위: `5-1 ~ 5-5`

### M7. 챕터 6 + 대장간
- 목표: 브란 장, 포대/파괴벽, 대장간/방어구 강화
- 챕터 범위: `6-1 ~ 6-5`

### M8. 챕터 7 + 문장 조율
- 목표: 시민 흐름 제어, 사리아, 무기군 지정 파밍
- 챕터 범위: `7-1 ~ 7-5`

### M9. 챕터 8 + 스텔스/암살 완성
- 목표: 레테, 표식/은신/분산 전투 완성
- 챕터 범위: `8-1 ~ 8-5`

### M10. 챕터 9A + 카일 합류
- 목표: 군기/검문/증언 삭제 구조, 카일 전환
- 챕터 범위: `9A-1 ~ 9A-5`

### M11. 챕터 9B + 선택 제작/옵션 교정
- 목표: 노아 합류, 전장 개정, 최종 기억, endgame crafting
- 챕터 범위: `9B-1 ~ 9B-5`

### M12. 챕터 10 + 엔딩 + 포스트게임
- 목표: 최종장, 일반/진엔딩, 최종 회상 토벌전
- 챕터 범위: `10-1 ~ 10-5`

- 각 장의 일반 상자 개수
- 편지 수
- 일부 유니크 무기 개수
- 2차 서브 목표 보상 수
- 중간 챕터의 hunt 보스 연출량

- bond assist 본격화
- suspend save
- postgame 최종 회상 토벌전 확장

## 15. 최종 메모

이 프로젝트의 제작은 “문서를 다 썼으니 이제 한 번에 만들자”가 아니다.  
오히려 반대다.

- 전투를 먼저 고정하고
- 캠프를 얹고
- 챕터를 하나씩 완성하고
- 그 위에 파밍과 성장 구조를 올리고
- 마지막에 엔딩과 포스트게임을 닫는 순서여야 한다

즉, 이 백로그의 핵심은 속도가 아니라 **붕괴하지 않는 순서**다.

이 순서만 지키면,
Codex로 구현해도 프로젝트는 점점 커지는 게 아니라
**점점 완성되는 방향**으로 간다.
