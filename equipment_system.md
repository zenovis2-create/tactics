# Equipment System Spec v1
## Project: 잿빛의 기억

## 1. 문서 목적

이 문서는 장비, 파밍, 강화, 분해, 문장, 제작, 옵션 교정까지 포함한 **장비 시스템 전체 규칙**을 고정하는 문서다.

이 문서가 고정하는 것은 아래 여섯 가지다.

1. 장비 카테고리와 슬롯 구조
2. 장비 획득 루트와 해금 타이밍
3. 무기 랜덤화 규칙과 희귀도 체계
4. 방어구 / 악세사리 / 소모품의 고정형 운용 규칙
5. 문장, 분해, 대장간, 선택 제작, 옵션 교정의 역할
6. 장비 시스템이 서사와 전투를 해치지 않게 하는 밸런스 원칙

이 문서는 `core_combat_spec.md`, `data_schema.md`, `boss_loot_tables.md`, `camp_ui_spec.md`와 함께 읽는다.

## 2. 핵심 설계 원칙

### 2-1. 랜덤성은 무기에 집중한다
이 게임의 랜덤 파밍은 기본적으로 **무기**만 담당한다.

- 무기: 랜덤 인스턴스
- 방어구: 고정형
- 악세사리: 고정형
- 소모품: 고정형

이렇게 해야 밸런스가 무너지지 않고, 각 장의 숨은 상자 보상도 서사적으로 의미가 남는다.

### 2-2. 스토리 첫 클리어는 실망시키지 않는다
보스를 잡았는데 쓰레기템이 나오면 안 된다.

- 첫 클리어: 최소 희귀 보장
- 최종 보스 첫 클리어: 최소 영웅 보장
- 회상 토벌전: 반복 가능 대신 최소 고급 보장

### 2-3. 장비는 수치보다 전술을 바꿔야 한다
좋은 장비는 단순히 공격력 +2가 아니라, 아래를 바꾼다.

- 숲에서 강해진다
- 망각 누적을 완화한다
- 표식 대응이 된다
- 레버/문/탑 조작이 쉬워진다
- 구조전에서 보호 가치가 생긴다
- 전장 개정에 저항한다

### 2-4. 슬롯 수는 적고 선택은 무겁게
유닛당 장비 슬롯은 3개로 고정한다.

- 무기 1
- 방어구 1
- 악세사리 1

이 구조 덕분에 수집은 많아도 실제 의사결정은 선명하다.

### 2-5. 초반에는 시스템보다 서사가 앞선다
장비 시스템은 점진적으로 열린다.

- 2장: 인벤토리, 악세사리
- 3장: 방어구, 첫 확정 보스 무기
- 4장: 첫 랜덤 보스 무기, 첫 회상 토벌전
- 5장: 분해, 문장 장부
- 6장: 대장간
- 7장: 문장 조율
- 9B: 선택 제작, 옵션 교정

### 2-6. 실패한 파밍에도 진척감이 남아야 한다
원하는 무기가 안 나와도 아래가 남는다.

- 보스 문장
- 분해 재료
- 유니크 중복 보상
- 무기군 지정
- 선택 제작

즉, “이번 판이 헛되지 않았다”는 느낌이 중요하다.

## 3. 장비 카테고리

## 3-1. 무기
랜덤 드롭의 핵심 장비.

### 무기 타입
- Sword
- Lance
- Bow
- Staff
- Tome

### 무기 역할
- 검: 근접 범용 / 균형형
- 창: 돌격 / 장교형 / 직선 압박
- 활: 원거리 / 표식 / 고립 처형
- 지팡이: 치유 / 정화 / 보호
- 마도서: 봉인 / 전장 간섭 / 원거리 압박

---

## 3-2. 방어구
고정형 장비. 랜덤 옵션 없음.

### 방어구 타입
- light
- heavy
- robe

### 역할
- light: 회피 / 기동 / 원거리 대응
- heavy: HP / DEF / 전열 유지
- robe: RES / 정화 / 지원 강화

---

## 3-3. 악세사리
고정형 장비. 탐색과 장별 전략의 핵심.

### 역할
- 망각 대응
- 표식 대응
- 지형 특화
- 구조전 특화
- 레버/상자/오브젝트 조작
- 특정 보스 기믹 카운터

---

## 3-4. 소모품
전투 중 사용하는 일회성 아이템.

예:
- 응급약
- 성수병
- 던지는 돌
- 정화약
- 해제향

---

## 3-5. 재료
강화/제작/교정에 쓰는 자원.

예:
- 금화
- 흑철편
- 기억분말
- 보스 문장

---

## 4. 슬롯 구조와 장착 규칙

## 4-1. 유닛당 슬롯
모든 플레이어블 유닛은 아래 슬롯을 가진다.

- `weapon_slot`
- `armor_slot`
- `accessory_slot`

### 예외
특수 NPC, 게스트, 보스는 실제 장착 UI 없이 템플릿 장비를 가질 수 있다.

---

## 4-2. 장착 제한
장착 가능 여부는 아래 순서로 판정한다.

1. 유닛 존재 여부
2. 유닛 현재 직업 확인
3. 직업의 `weapon_types_allowed` / `armor_types_allowed` 확인
4. 아이템 특수 제한 확인
5. 고정 장비 여부 확인

### 예
- 브란은 heavy armor 가능
- 세린은 robe만 가능
- 티아는 bow, light armor 위주
- 노아는 staff 전용 또는 제한적 Tome 허용 가능

---

## 4-3. 장착 교체 가능 시점
기본적으로 아래에서만 장착 변경 가능하다.

- 캠프
- 출격 준비 화면

전투 중 장착 변경은 허용하지 않는다.  
전투 중에는 소모품만 사용 가능하다.

---

## 5. 장비 능력치 적용 순서

최종 능력치는 아래 순서로 계산한다.

1. `UnitData.base_stats`
2. `ClassData` 보정 / 클래스 패시브
3. 장착 무기 기본 수치
4. 장착 방어구 고정 보너스
5. 장착 악세사리 고정 보너스
6. 무기 인스턴스 어픽스 보너스
7. 강화 수치
8. 전투 중 버프 / 상태이상 / 지형 / 필드 효과

### 핵심 원칙
- 무기만 랜덤 인스턴스 수치를 가진다
- 방어구와 악세사리는 기본적으로 템플릿 그대로 적용된다
- 장비 수치 계산은 UI가 아니라 `EquipmentService`가 담당한다

## 6. 무기 시스템

## 6-1. 무기 구성
무기는 아래 3층으로 나뉜다.

### 1) Base Weapon
고정 템플릿.  
무기 타입, 기본 위력, 기본 명중, 기본 치명, 사거리, 기본 성향을 가진다.

### 2) Affix
랜덤 옵션.  
Prefix, Suffix, Neutral 형태로 붙을 수 있다.

### 3) Weapon Instance
실제 플레이어 인벤토리에 들어가는 개별 무기다.

즉:

```text
WeaponInstance = Base Weapon + Affix Set + Rolled Values
```

7. 방어구 시스템
7-1. 방어구는 고정형이다

방어구는 랜덤이 아니다.
이유는 세 가지다.

전열/후열 밸런스를 망치기 쉽다
탐색 보상과 중복되기 쉽다
랜덤성은 무기에 집중하는 편이 플레이 체감이 좋다
7-2. 방어구 강화

방어구는 대장간에서 강화한다.

8. 악세사리 시스템
8-1. 악세사리는 장별 전략 장비다

악세사리는 수치보다 기능을 우선한다.

예:

표식 대상이 되면 첫 추격 피해 감소
레버 상호작용이 행동 종료를 일으키지 않음
규칙 개정 첫 1회 무효

8-3. 악세사리 슬롯 전략

10장 최종전 → 공명 매듭 또는 월식의 인장

19. 밸런스 규칙
19-1. 초반 파워 억제

3~4장 장비가 8장 장비처럼 느껴지면 안 된다.

금지
초반에 광역 디버프 완전 무효
초반에 표식 완전 면역
초반에 공명/앵커 관련 최종장급 효과
19-2. 역할 전도 금지

장비가 캐릭터 역할을 완전히 바꾸면 안 된다.

예:

브란이 회피 암살형이 되면 안 됨
세린이 순수 딜러가 되면 안 됨
티아가 heavy tank처럼 굴면 안 됨

장비는 역할을 넓히되 정체성은 유지해야 한다.

19-3. 힐/정화 과증폭 금지

정화봉, 악세사리, 지형 보정이 겹쳐
망각이 의미 없는 상태가 되면 안 된다.

19-4. 최종장 밸런스

10장 전용 장비는 강해도 되지만,
이름 앵커 / 공명 / 동료 공명 인장보다 우위에 서면 안 된다.
즉 장비가 최종전의 내러티브 기믹을 대체하면 안 된다.

22. 최종 메모

이 장비 시스템의 목표는 플레이어를 숫자 표에 가두는 게 아니다.
목표는 아래 감각을 만드는 것이다.

상자를 열면 전술이 달라진다
보스를 잡으면 그 철학이 손에 남는다
원하는 무기를 못 먹어도 다음 시도가 무의미하지 않다
최종적으로는 “내 파티의 기억과 역할”에 맞는 장비 세팅이 완성된다

즉, 이 시스템은 파밍 게임이 아니라
서사와 전투를 붙잡아 주는 수집/성장 시스템이어야 한다.
