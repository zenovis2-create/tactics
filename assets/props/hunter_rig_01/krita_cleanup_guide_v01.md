# Hunter Rig 01 Krita Cleanup Guide V01

## Purpose

This guide converts the current hunter-rig image into a reusable runtime-facing chapter prop.

## Expected Source

When a chosen hunter-rig candidate is placed in:

- `source/`

use this guide to produce the clean and runtime outputs.

## Cleanup Goal

Produce a clean forest-device asset that:

- has transparent background
- preserves stakes / trigger / line separation
- stays suspicious-device-first and clutter-second
- can support chapter map previews, landmark cards, and icon surfaces

## Step 1. Remove Background

- keep full transparency
- remove all paper, glow, and framing
- do not keep drop-shadow haze beyond a minimal contact-softening edge

## Step 2. Preserve Major Structure

Keep:

- readable stake anchors
- visible trigger or tension element
- clear device direction
- one restrained accent only

Do not add:

- giant spikes
- blade clusters
- decorative twig clutter

## Step 3. Tighten Value Separation

- stakes must stay distinct from rope or line
- trigger logic must read before the accent
- accent must stay tertiary

## Step 4. Prepare Reusable Output

The cleaned rig should support:

- chapter landmark sheet
- map integration surface
- future icon extraction

That means the silhouette must remain clean and centered.

## Step 5. Export Clean Output

Export:

- `hunter_rig_01_clean_v01.png`

Store it under:

- `/Volumes/AI/tactics/assets/props/hunter_rig_01/clean/`

## Step 6. Prepare Runtime Derivatives

Current derivatives:

- `hunter_rig_01_landmark_v01.png`
- `hunter_rig_01_icon_v01.png`
- `hunter_rig_01_integration_v01.png`

Store these under:

- `/Volumes/AI/tactics/assets/props/hunter_rig_01/runtime/`

## Acceptance Checklist

- background fully removed
- stakes, trigger, and line remain distinct
- silhouette survives 128px reduction
- reads as hunter rig, not lever or branch pile
- sits in the same world as Greenwood tiles and CH03 ambush landmarks
