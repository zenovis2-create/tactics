# Scavenged Cache 01 Krita Cleanup Guide V01

## Purpose

This guide converts the current scavenged-cache image into a reusable runtime-facing chapter prop.

## Expected Source

When a chosen scavenged-cache candidate is placed in:

- `source/`

use this guide to produce the clean and runtime outputs.

## Cleanup Goal

Produce a clean investigation prop that:

- has transparent background
- preserves crate / lid / wrap separation
- stays cache-first and reward-second
- can support chapter map previews, investigation cards, and icon surfaces

## Step 1. Remove Background

- keep full transparency
- remove all paper, glow, and framing
- do not keep drop-shadow haze beyond a minimal contact-softening edge

## Step 2. Preserve Major Structure

Keep:

- readable crate or chest body
- visible opening plane
- strap or wrap logic
- one restrained accent only

Do not add:

- treasure aura
- loose item spill
- heavy metal corner decoration

## Step 3. Tighten Value Separation

- wood body must stay distinct from wrap
- opening plane must read before the accent
- accent must stay tertiary

## Step 4. Prepare Reusable Output

The cleaned cache should support:

- chapter prop sheet
- map integration surface
- future icon extraction

That means the silhouette must remain clean and centered.

## Step 5. Export Clean Output

Export:

- `scavenged_cache_01_clean_v01.png`

Store it under:

- `/Volumes/AI/tactics/assets/props/scavenged_cache_01/clean/`

## Step 6. Prepare Runtime Derivatives

Current derivatives:

- `scavenged_cache_01_object_v01.png`
- `scavenged_cache_01_icon_v01.png`
- `scavenged_cache_01_integration_v01.png`

Store these under:

- `/Volumes/AI/tactics/assets/props/scavenged_cache_01/runtime/`

## Acceptance Checklist

- background fully removed
- body, opening plane, and wrap remain distinct
- silhouette survives 128px reduction
- reads as searchable cache, not altar or treasure chest
- sits in the same world as CH01 props and current investigation landmarks
