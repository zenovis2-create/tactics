# Revision Core 01 Krita Cleanup Guide V01

## Purpose

This guide converts the current revision-core image into a reusable runtime-facing chapter prop.

## Expected Source

When a chosen revision-core candidate is placed in:

- `source/`

use this guide to produce the clean and runtime outputs.

## Cleanup Goal

Produce a clean root-archive landmark asset that:

- has transparent background
- preserves core / frame / accent separation
- stays revision-pressure-first and altar-second
- can support chapter map previews, landmark cards, and icon surfaces

## Step 1. Remove Background

- keep full transparency
- remove all paper, glow, and framing
- do not keep drop-shadow haze beyond a minimal contact-softening edge

## Step 2. Preserve Major Structure

Keep:

- readable central core
- visible outer frame
- clear stabilizer mass
- one restrained accent only

Do not add:

- crystal spires
- altar wings
- pipe clutter

## Step 3. Tighten Value Separation

- core must stay distinct from the frame
- frame must read before the accent
- accent must stay tertiary

## Step 4. Prepare Reusable Output

The cleaned revision core should support:

- chapter landmark sheet
- map integration surface
- future icon extraction

That means the silhouette must remain clean and centered.

## Step 5. Export Clean Output

Export:

- `revision_core_01_clean_v01.png`

Store it under:

- `/Volumes/AI/tactics/assets/props/revision_core_01/clean/`

## Step 6. Prepare Runtime Derivatives

Current derivatives:

- `revision_core_01_landmark_v01.png`
- `revision_core_01_icon_v01.png`
- `revision_core_01_integration_v01.png`

Store these under:

- `/Volumes/AI/tactics/assets/props/revision_core_01/runtime/`

## Acceptance Checklist

- background fully removed
- core, frame, and accent remain distinct
- silhouette survives 128px reduction
- reads as revision core, not altar or generic machine clutter
- sits in the same world as CH09B root-archive and revision-pressure landmarks
