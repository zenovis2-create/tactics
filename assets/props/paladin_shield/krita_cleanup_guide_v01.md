# Paladin Shield Krita Cleanup Guide V01

## Purpose

This guide converts the current shield concept direction into a runtime-friendly equipment-support asset.

## Expected Source

When a selected shield candidate is placed in:

- `source/`

use this guide to produce a clean equipment-support output.

## Cleanup Goal

Produce a clean shield asset that:

- has transparent background
- preserves the broad silhouette and thick rim
- keeps one focal emblem area only
- can support loadout sheets, UI extraction, and integration previews

## Step 1. Remove Background

- delete the white or paper-like background completely
- keep only minimal contact shadow if needed
- avoid leaving faded border haze around the shield

## Step 2. Preserve Major Structure

Keep:

- outer rim
- main shield plate
- one central focal emblem or accent zone

Do not add:

- secondary trim rings
- tiny decorative relief
- lace-like ornament

## Step 3. Tighten Value Separation

- rim must read separately from the main plate
- leather support should remain secondary
- accent must not become louder than the structure

## Step 4. Prepare Reusable Output

The cleaned shield should work in more than one context:

- equipment reference sheet
- loadout support art
- future icon extraction

That means the contour must remain simple and stable.

## Step 5. Export Runtime Candidate

Export:

- `paladin_shield_clean_v01.png`

Store it under:

- `/Volumes/AI/tactics/assets/props/paladin_shield/clean/`

## Step 6. Prepare Runtime Derivatives

Later exports may include:

- `paladin_shield_equipment_v01.png`
- `paladin_shield_icon_v01.png`
- `paladin_shield_integration_v01.png`

Store these under:

- `/Volumes/AI/tactics/assets/props/paladin_shield/runtime/`

## Acceptance Checklist

- background fully removed
- rim remains thick and readable
- central emblem stays simple
- shield reads as defense before decoration
- output feels like the same world as the current knight and heavy-class lane
