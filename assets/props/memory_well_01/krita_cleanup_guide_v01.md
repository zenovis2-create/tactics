# Memory Well 01 Krita Cleanup Guide V01

## Purpose

This guide converts the current memory-well landmark image into a reusable runtime-facing chapter prop.

## Expected Source

When a chosen memory-well candidate is placed in:

- `source/`

use this guide to produce the clean and runtime outputs.

## Cleanup Goal

Produce a clean landmark asset that:

- has transparent background
- preserves rim / opening / support-frame separation
- stays well-first and memory-second
- can support chapter map previews, landmark cards, and icon surfaces

## Step 1. Remove Background

- keep full transparency
- remove all paper, glow, and framing
- do not keep drop-shadow haze beyond a minimal contact-softening edge

## Step 2. Preserve Major Structure

Keep:

- readable well rim
- visible shaft opening
- upright support frame
- one restrained accent only

Do not add:

- shrine candles
- giant spectral effects
- clutter piles

## Step 3. Tighten Value Separation

- stone rim must stay distinct from the support frame
- opening must read before the accent
- accent must stay tertiary

## Step 4. Prepare Reusable Output

The cleaned well should support:

- chapter landmark sheet
- map integration surface
- future icon extraction

That means the silhouette must remain clean and centered.

## Step 5. Export Clean Output

Export:

- `memory_well_01_clean_v01.png`

Store it under:

- `/Volumes/AI/tactics/assets/props/memory_well_01/clean/`

## Step 6. Prepare Runtime Derivatives

Current derivatives:

- `memory_well_01_landmark_v01.png`
- `memory_well_01_icon_v01.png`
- `memory_well_01_integration_v01.png`

Store these under:

- `/Volumes/AI/tactics/assets/props/memory_well_01/runtime/`

## Acceptance Checklist

- background fully removed
- rim, opening, and support remain distinct
- silhouette survives 128px reduction
- reads as well landmark, not altar or rubble
- sits in the same world as CH01 props and current objective families
