# CH04 Sacred Machinery Preview V01

## Purpose

This preview exists to validate the mixed-language object problem for CH04-like spaces:

- sacred objective language
- local mechanism language
- route/system-control language

It asks one question:

`Can altar, lever, and gate-control coexist inside one flooded-monastery style surface without collapsing into one interaction family?`

## Required Ingredients

The frame should include:

- Serin
- Rian
- Tia
- Enemy Raider
- Enemy Skirmisher
- fortress-like stone ground as current stand-in for engineered sacred floor
- Altar 01
- Lever 01
- Gate Control 01

## Read Order

The intended read order is:

1. allied formation
2. hostile pressure
3. sacred focal object
4. local mechanical object
5. route/system-control object
6. ground family

If machinery and sacred objects merge into one category on first read, the frame fails.

## Pass Conditions

- altar still reads sacred before mechanical
- lever still reads local and hand-operated
- gate control still reads infrastructural and larger-system oriented
- Serin remains readable over the same floor family as Rian and Tia
- hostile units remain distinct from allies despite the denser environment

## Current Recommendation

Use this preview as a decision frame before:

- creating a dedicated sacred-floor terrain family
- promoting additional monastery object families
- routing altar/lever/gate-control into chapter-specific map kits
