# Chapter 1 M2 QA Gate (2026-04-13)

## Scope

This QA gate applies only to the active Chapter 1 M2 shell:

- `CH01_02` through `CH01_05`
- battle to cutscene to camp progression
- CampHub v1
- memory / evidence / letter read-only views
- regression smoke against the validated M1 battle loop

Out of scope:

- save/load
- equipment meta
- monetization / IAP / store assets
- random drops / hunt systems
- later chapters

## Must-Pass Automated Checks

These checks must all pass on the candidate:

- `bash scripts/dev/check_runnable_gate0.sh`
- `godot4 --headless --path . --quit`
- `godot4 --headless --path . --script res://scripts/dev/m1_core_loop_contract_runner.gd`
- `godot4 --headless --path . --script res://scripts/dev/m1_playtest_runner.gd`
- `godot4 --headless --path . --script res://scripts/dev/m2_campaign_flow_runner.gd`
- `godot4 --headless --path . --script res://scripts/dev/m3_ui_runner.gd`

## Must-Pass Functional Checklist

### 1. Battle runtime regression

- Battle scene opens without script error.
- One full battle can still finish from start to victory.
- Select -> move -> attack or wait still works.
- Cancel still restores the moved unit to its original tile.
- Counterattack still resolves exactly once when valid.
- Enemy phase still resolves without illegal move-attack behavior.

### 2. Chapter 1 progression

- `CH01_02` starts first.
- Clearing `CH01_02`, `CH01_03`, and `CH01_04` routes into cutscene handoff.
- Advancing from cutscene enters the next stage in order.
- Clearing `CH01_05` routes into camp instead of the next battle.
- Camp can advance into the Chapter 1 completion state.

### 3. CampHub v1

- Camp panel opens in `camp` mode.
- Recommendation text is non-empty.
- `summary`, `party`, `inventory`, and `records` sections all exist.
- Party roster is non-empty.
- Selecting a different party member updates the selected-party detail view.
- Inventory list is non-empty after the Chapter 1 flow.

### 4. Records surfaces

- Memory list is non-empty in camp.
- Evidence list is non-empty in camp.
- Letter list is non-empty in camp.
- Records tab is the recommended landing section for the Chapter 1 camp handoff.

### 5. Camp readability cues

- Party tab shows a readiness-style badge.
- Inventory tab shows an item-count badge.
- Records tab shows a `NEW` badge with count.
- Alert line clearly signals new records / letter / camp readiness.

## Manual Smoke Checklist

Run these manually in the editor or desktop build before calling the shell demo-ready:

1. Play through one battle and confirm touch-sized controls remain usable.
2. Open the battle inventory panel and verify world input is blocked while it is open.
3. Clear Chapter 1 in order and read the cutscene handoff text after each battle.
4. Enter camp after `CH01_05` and verify the player can understand:
   - what memory was recovered
   - what evidence was recovered
   - that Serin has formally joined
   - where the next destination points
5. Switch between `party`, `inventory`, and `records` and confirm the tab labels and badges still make sense.

## Exit Criteria

The Chapter 1 M2 shell is ready for the next milestone only if:

- all automated checks pass
- no reproducible crash, softlock, or broken transition remains
- no missing `party`, `inventory`, or `records` state appears in camp
- the current camp recommendation correctly points the player to the intended next review surface
- no out-of-scope M2 systems were added to satisfy the above
