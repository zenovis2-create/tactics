## Comparison Goal

The player should be able to read these objects before UI help as:

- memory well: investigation landmark with narrative disturbance weight
- scavenged cache: searchable civilian stash and quiet reward point
- altar: sacred objective
- lever: local mechanism
- gate control: route or system control

If CH01 props collapse into the same category as altar or lever, chapter-local clarity has failed.

## Current Read Check

### Memory Well

- First read: chapter landmark, searchable ruin well
- Must stay distinct from: altar and generic rubble
- Drift risk: reading like a shrine because of accent overuse

### Scavenged Cache

- First read: searchable hidden stash
- Must stay distinct from: treasure chest and military crate
- Drift risk: reading like reward loot instead of investigation prop

### Altar

- First read: sacred objective
- Must stay distinct from: chapter-local ruined-village landmarks
- Drift risk: overusing altar where quieter local props should carry the beat

### Lever

- First read: local mechanism
- Must stay distinct from: chapter landmark or narrative anchor
- Drift risk: becoming the focal object of a scene that should belong to the map instead

### Gate Control

- First read: infrastructural route logic
- Must stay distinct from: lever and well
- Drift risk: shrinking into a generic control box

## Current Conclusion

CH01 now has a usable chapter-local prop split:

- `memory_well_01` for investigation plus memory disturbance
- `scavenged_cache_01` for quiet search and recovery beats

This allows altar, lever, and gate control to keep their broader project meanings instead of doing all early-game interaction work alone.
