# Equipment Icon Visual Pack

## Purpose

This document defines the in-game equipment icon direction for `잿빛의 기억`.
It converts the approved portrait baseline into a usable icon language for weapons, armor, and accessories.

This is an in-game production pack.
It is not a storefront art brief and it is not a lore document.

## Direction Lock

`Retro tactical equipment icon sheet`

- Icons should feel like they belong beside the accepted portrait baseline in `/Volumes/AI/tactics/artifacts/ash15/ash15_portrait_sheet_v3.png`.
- The target mood is classic console tactics RPG menu art:
  - simple read
  - warm parchment-friendly contrast
  - controlled cel-style planes
  - no glossy fantasy loot-card rendering

## Core Principles

1. `Read at mobile scale first`
2. `Class identity before ornament`
3. `Record / seal / map grammar stays visible`
4. `A single icon family across weapon, armor, and accessory slots`

## Render Rules

- Use clean, even line weight.
- Use flatter cel-shaded fills rather than painterly texture.
- Keep highlights restrained and matte.
- Avoid hyper-detailed metal scratches or gemstone shine.
- Do not render icons like splash-art inventory cards.

## Palette Rules

- Base neutrals: ash iron, brown leather, parchment bone.
- Ally emphasis: controlled vow gold or pale blue accents.
- Ranger/forest gear: moss green used sparingly.
- Threat or corrupted variants: ember red only where gameplay-relevant.
- Do not rely on saturated rainbow rarity colors.

## Shape Rules

### Weapons

- Each weapon type needs one unmistakable silhouette:
  - sword: straight or slightly tapered blade with readable guard
  - lance: long thrusting line and narrow head
  - bow: clear arc with visible grip break
  - staff: vertical shaft with one readable head motif

### Armor

- Armor should read as worn field gear, not parade costume.
- Heavy armor uses broad plate masses and sturdy strap logic.
- Light armor uses trimmed silhouettes and layered cloth/leather planes.
- Robes use vertical fall and sacred trim, not oversized magic effects.

### Accessories

- Central symbol plus one secondary detail.
- Use seals, rings, tabs, badges, clasps, marks, knots, and shards.
- Accessories should feel tied to memory, command, archives, or vows.

## Slot Family Rules

- `Weapon`, `Armor`, and `Accessory` should feel like one menu family.
- They may vary by shape and mass, but not by art style.
- No slot should look more modern or more painterly than the others.

## Production Sheet Format

- Preferred output: one clean icon concept sheet.
- Group by:
  - weapon set
  - armor set
  - accessory set
- Use neutral parchment or muted panel backing.
- No baked-in labels unless the sheet is explicitly for internal review markup.

## Prompting Rules

- Ask for `classic tactical JRPG equipment icon sheet`.
- Ask for `clean linework`, `slightly flatter cel shading`, and `retro menu-art readability`.
- Forbid:
  - glossy loot-card polish
  - painterly item-card rendering
  - neon rarity glows
  - western dark-fantasy grime overload

## Initial Production Scope

- CH02~CH04 accessory line cleanup against the portrait baseline
- core weapon icons for sword, lance, bow, staff
- core armor icons for heavy, light, robe
