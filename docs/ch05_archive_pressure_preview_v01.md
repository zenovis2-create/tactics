# CH05 Archive Pressure Preview V01

## Purpose

This preview validates whether the current interaction-object family still reads correctly inside a Gray Archive pressure surface.

It asks:

`Can altar, lever, and gate-control remain distinct when the chapter context shifts from fortress machinery into sealed archive pressure?`

## Required Ingredients

The frame should include:

- `fortress_tile_01`
- `fortress_tile_02`
- `altar_01`
- `lever_01`
- `gate_control_01`
- `field_sword_01`
- `Serin`
- `Rian`
- `Tia`
- `Enemy Raider`
- `Enemy Skirmisher`

## Review Focus

Compared to the fortress-family previews, this preview should push:

- archive seal pressure
- evidence-first sacred focus
- mechanical controls embedded in shelf / route logic
- lower openness and more aisle-like pressure

## Intended Read Order

1. player route and frontline pressure
2. truth-bearing anchor
3. local mechanism
4. route/system-control device
5. archive floor family
6. support equipment surface

If machinery becomes the first read before the units and the truth anchor, the preview fails.

## Pass Conditions

- `altar_01` still reads as sacred / truth-bearing rather than generic archive clutter
- `lever_01` still reads as local mechanism
- `gate_control_01` still reads as larger route/system control
- the floor family does not overpower the roster
- `field_sword_01` can sit in the frame as support equipment without becoming the focal object

## Working Goal

This preview is the first archive-context proof point for the now runtime-routed interaction family.

If it passes, the object-family split is no longer fortress-only.
