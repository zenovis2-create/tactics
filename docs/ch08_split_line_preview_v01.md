# CH08 Split-Line Preview V01

## Purpose

This preview validates whether the locked interaction-object family still separates correctly inside a split-line pursuit surface.

It asks:

`Can altar, lever, and gate-control stay legible when the map becomes a pursuit lane with divided route comfort instead of fortress or archive pressure?`

## Required Ingredients

The frame should include:

- `forest_tile_01`
- `forest_tile_02`
- `altar_01`
- `lever_01`
- `gate_control_01`
- `field_sword_01`
- `Rian`
- `Tia`
- `Enemy Raider`
- `Enemy Skirmisher`

## Read Targets

- `altar`: pursuit mark / memory anchor / ruin shrine
- `lever`: local hunter-device or release trigger
- `gate_control`: engineered remnant that changes route state

## Pass Conditions

- the surface still reads as divided pursuit space rather than fortress reuse
- altar does not collapse into generic forest clutter
- lever still reads as local and immediate
- gate-control still reads as larger route logic
- units remain the primary screen read
