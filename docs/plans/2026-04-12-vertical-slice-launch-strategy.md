# Vertical Slice Launch Strategy: External Positioning, Campaign Priorities, and Storefront Expectations

## Summary

This document sets the publishing strategy for the first public-facing phase of the project and the path from vertical-slice reveal to launch readiness.

Primary outcomes:

- lock external framing before public reveal
- align campaign sequencing across marketing, trailer, and storefront work
- define what can be shown now vs what remains internal
- set storefront quality and conversion expectations for Steam, App Store, and Google Play

## 1. Launch Strategy

Use a two-horizon launch model.

### Horizon A: Vertical Slice Reveal (near-term)

Goal:

- prove concept-market fit and generate early wishlist/follow intent without overpromising content scale

Reveal shape:

1. Controlled reveal announcement (teaser + short positioning line)
2. Vertical-slice gameplay reveal (core loop + memory mechanic proof)
3. Storefront page activation and iterative optimization

### Horizon B: Full Launch (later)

Goal:

- convert validated positioning into broad acquisition and launch-week conversion

Launch shape:

1. pre-launch campaign escalation (creator, press, feature explainers)
2. launch window conversion push
3. post-launch trust loop (patch notes, responsiveness, retention beats)

## 2. External Messaging

## 2.1 External Positioning (locked)

`A mobile-first dark fantasy tactical RPG where each short battle is about protecting your squad's memories as much as winning the map.`

## 2.2 Message Pillars (public)

1. Tactics-first, mobile-readable combat
2. Memory as gameplay pressure, not just lore flavor
3. Squad bonds that have combat consequence
4. Melancholic war narrative with meaningful choices

## 2.3 Messaging Boundaries (must enforce)

Public-safe now:

- amnesiac commander setup
- companionship beginning with Serin
- memory fragments as chapter reward and gameplay unlock context
- chapter one promise: protect people first, identity answers later

Internal-only until later beats:

- `제로` real-name reveal framing
- Karuon relationship truth framing
- mechanism-level truth of `무명의 종`
- late-game philosophy twists and ending-state specifics

## 2.4 Voice Rules

- calm, direct, specific
- no hype inflation about scope or timeline
- no gacha/live-service framing
- no revenge-hero framing

## 3. Reveal Timing and Gates

`D` = vertical-slice reveal day.

### D-21 to D-14: Narrative/Positioning Lock

- confirm canon-safe messaging and spoiler boundaries
- lock one-sentence positioning and message pillars
- publish internal FAQ for all outward copy owners

### D-14 to D-7: Asset and Store Pack Lock

- finalize storefront metadata variants
- finalize teaser (30s) and trailer (60s) beat sheets

### D-7 to D-1: Dry Run and Safety Check

- run end-to-end publish rehearsal

### D-Day: Controlled Reveal

- release teaser + key art + primary positioning line
- publish storefront page copy and screenshots

### D+1 to D+14: Optimize and Scale

- tune thumbnails, ordering, and first-paragraph copy based on CTR/retention signals
- prioritize one tactical-clarity proof asset and one narrative-stakes proof asset per week
- expand creator outreach once conversion baseline is stable

## 5. Storefront Expectations

## 5.1 Shared Expectations (all platforms)

- first screen must communicate: tactical readability + memory theme + squad stakes
- first paragraph must define genre, session feel, and unique mechanic
- screenshots must alternate between system clarity and emotional context
- metadata must avoid misleading scale claims and spoiler leakage

## 5.2 Steam Expectations

- store capsule/hero art: commander + squad + fractured-memory motif
- short description: one clear tactical promise + one thematic differentiator
- tags: tactics RPG + narrative + single-player and adjacent fit tags
- discovery objective: maximize wishlist conversion from first visit

## 5.3 App Store Expectations

- concise, high-signal name/subtitle strategy
- search keywords focused on tactical intent and fantasy context
- no keyword stuffing or competitor-trademark behavior
- preview and screenshot ordering tuned for readability-first mobile buyers

## 5.4 Google Play Expectations

- short description optimized for immediate clarity and differentiation
- full description structured by: core loop -> memory mechanic -> squad stakes -> chapter promise
- metadata cleanliness and policy-safe promotional language
- screenshot set optimized for mobile tactical UI readability

## 7. Success Metrics

Reveal-phase success metrics:

- messaging consistency: no cross-surface contradiction in core promise
- storefront conversion baseline: improving page-to-wishlist/follow trend week-over-week
- asset performance: teaser completion and key-art click-through trend up after first iteration
- trust signal: community feedback reflects clarity on "what game this is" and "what is not yet promised"

## 8. Operating Cadence

- weekly publishing sync: positioning drift check + storefront signal review
- weekly dependency review with Campaign/Storefront/Content owners
- pre-reveal sign-off required from Publishing Director before any public release
