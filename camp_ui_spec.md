# Camp UI Spec v1
## Project: 잿빛의 기억

## 1. 문서 목적

이 문서는 캠프 허브, 장비 화면, 인벤토리, 분해, 대장간, hunt 보드, 문장 조율, 선택 제작, 옵션 교정, 기억/물증/편지 확인까지 포함한 **캠프 UI 전체 구조**를 고정하는 문서다.

이 문서가 고정하는 것은 아래 여섯 가지다.

1. 캠프의 정보 구조와 화면 흐름
2. 모바일 우선 입력 원칙
3. 장비/인벤토리/분해/강화/파밍 화면의 기능 범위
4. 캠프 진입 시 어떤 콘텐츠를 어떤 우선순위로 보여 줄지
5. 저장 데이터와 UI 상태의 연결 방식
6. UI가 시스템 로직을 중복 구현하지 않도록 하는 규칙

이 문서는 `equipment_system.md`, `data_schema.md`, `flag_progression_spec.md`, `production_backlog.md`와 함께 읽는다.

## 2. 핵심 설계 원칙

### 2-1. 모바일 우선, 탭 중심
캠프 UI는 기본적으로 **탭 / 클릭 중심**으로 설계한다.

- 드래그 앤 드롭 사용 금지
- 긴 목록은 바텀시트/슬라이딩 패널 사용
- 한 화면에 너무 많은 정보 동시 표시 금지
- 중요한 액션은 한 번에 한 단계만

### 2-2. 정보는 많아도 구조는 단순해야 한다
캠프가 후반에 커지더라도 플레이어가 길을 잃으면 안 된다.  
따라서 메인 허브에서 갈 수 있는 큰 축은 아래로 제한한다.

- 편성
- 장비
- 창고
- 분해
- 회상 토벌
- 대장간
- 기록(기억 / 물증 / 편지)

### 2-3. UI는 계산하지 않고 표시한다
장비 가능 여부, 최종 수치 계산, 드롭 생성, 문장 소비 검증은 전부 서비스 계층에서 한다.  
UI는 그 결과를 보여 주고 요청만 보낸다.

### 2-4. “지금 해야 할 것”을 강하게 보여 준다
캠프는 단순한 메뉴 묶음이 아니라, 장과 장 사이의 리듬 장치다.  
따라서 매 캠프 진입 시 플레이어가 바로 이해해야 한다.

- 다음 목적지는 어디인가
- 새로 얻은 게 무엇인가
- 누가 합류했는가
- 지금 해금된 기능은 무엇인가
- 당장 볼 만한 대화/편지가 있는가

### 2-5. 장비와 서사가 분리되지 않게 한다
예를 들어 7장 직후엔 미라/네리 편지가 보이고,  
8장 직후엔 레테 드롭이 추천되고,  
9B 직후엔 최종 기억 복원과 선택 제작이 자연스럽게 이어져야 한다.

즉, 캠프는 메뉴 허브이면서 동시에 **서사 허브**다.

---

## 3. 캠프 진입 흐름

스테이지 클리어 후 캠프 진입 순서는 아래로 고정한다.

1. 클리어 보상 팝업
2. 기억 조각 / 물증 획득 연출
3. 영입 연출(있다면)
4. 시스템 해금 팝업(있다면)
5. 캠프 진입
6. 우선순위 높은 공명 대화 / 편지 노출
7. 허브 자유 탐색 가능

### 이유
보상과 스토리 정보가 먼저 정리된 뒤 캠프에 들어가야,  
플레이어가 허브에서 무엇을 해야 하는지 이해하기 쉽다.

---

## 4. 캠프 허브 정보 구조

캠프 메인 화면의 1차 메뉴는 아래로 고정한다.

- **편성**
- **장비**
- **창고**
- **분해**
- **회상 토벌**
- **대장간**
- **기록**

### 4-1. 해금 순서
- 편성: 1장
- 장비: 2장
- 창고: 2장
- 분해: 5장
- 회상 토벌: 4장
- 대장간: 6장
- 기록: 1장부터 기본

해금 전 메뉴는 아래 방식 중 하나를 쓴다.

- 완전 숨김
- 잠금 아이콘 + 해금 조건 툴팁

초기 구현에서는 **숨김**이 더 단순하고 좋다.

## 5. 캠프 허브 화면 설계

## 5-1. 상단 바
항상 표시하는 정보

- 금화
- 흑철편
- 기억분말
- 문장 총합 요약
- 새 편지/대화 배지
- 저장 상태 아이콘

### 원칙
상단 바는 어떤 하위 화면으로 들어가도 유지하거나,  
최소한 같은 정보를 같은 위치에 보여 줘야 한다.

---

## 5-2. 중앙 허브 카드
큰 버튼 카드 형태를 권장한다.

예:
- 편성
- 장비
- 회상 토벌
- 기록

후반 해금 메뉴는 2차 줄 또는 하단 탭으로 보조한다.

### 카드에 표시할 수 있는 것
- `NEW`
- `!`
- 잠금
- 추천
- 진행 중
- 새로운 합류 동료 얼굴

예:
- 회상 토벌 카드에 `NEW` 배지
- 기록 카드에 편지 1개 도착 표시

---

## 5-3. 하단 패널
캠프에 들어오면 하단 패널에 아래 중 하나를 우선 노출한다.

우선순위:
1. 방금 해금된 시스템
2. 방금 획득한 물증 관련 공명 대화
3. 방금 도착한 편지
4. 다음 목적지 힌트

이 패널은 닫을 수 있어야 한다.

## 6. 편성 화면

## 6-1. 목적
- 출전 멤버 선택
- 현재 사용 가능한 동료 확인
- 직업/역할/장비 상태 요약 확인

## 6-2. 기본 레이아웃
왼쪽 또는 상단:
- 현재 파티 슬롯

오른쪽 또는 하단:
- 보유 동료 목록

선택 시 보여 줄 정보:
- 이름
- 직업
- 역할 태그
- 장비 요약
- 현재 레벨
- 최근 획득/합류 표시
- 주요 공명 인장 보유 여부(후반)

## 6-3. 조작
- 캐릭터 카드 탭 → 상세
- 파티 슬롯 탭 → 교체 후보 목록 열기
- 길게 누르기 → 상세 프로필
- 출격 준비 화면에서도 같은 컴포넌트 재사용

## 6-4. MVP 범위
초기에는 아래만 있어도 충분하다.

- 합류 여부 표시
- available 여부 표시
- 파티 포함/제외
- 장비 화면으로 이동 버튼

## 7. 장비 화면

## 7-1. 목적
- 무기/방어구/악세사리 장착/해제
- 장비 비교
- 현재 스탯 확인
- 역할에 맞는 추천 판단

## 7-2. 기본 흐름
캠프 → 장비 → 유닛 선택 → 슬롯 선택 → 아이템 목록 → 상세 비교 → 장착

## 7-3. 레이아웃
왼쪽:
- 캐릭터 초상
- 이름
- 직업
- 역할 태그

가운데:
- 무기 슬롯
- 방어구 슬롯
- 악세사리 슬롯

오른쪽:
- 최종 스탯
  - HP
  - ATK
  - DEF
  - RES
  - HIT
  - AVO
  - CRIT
  - MOVE
  - 사거리

하단:
- 아이템 목록 바텀시트

## 7-4. 비교 패널
아이템 하나를 탭하면 아래가 떠야 한다.

- 현재 장비와 비교
- 증감 수치
- 특수 효과
- 장착 가능 여부
- 추천 캐릭터 힌트

예:
```text
ATK 18 → 20 (+2)
HIT 80 → 88 (+8)
효과: 표식 대상 피해 +10%
티아에게 적합
```

19. 추천 표시 시스템

캠프 UI는 플레이어에게 너무 많은 선택지를 던지기보다,
현재 맥락상 중요한 걸 약하게 추천해 줘야 한다.

추천 예시
“이 스테이지는 망각 압박이 강합니다”
“레테 hunt에는 표식 대응 장비가 유리합니다”
“사리아 문장 5개가 모였습니다. 무기군 조율 가능”
“도른이 새 강화 작업을 제안합니다”

추천은 강제가 아니라 힌트여야 한다.

23. 최종 메모

캠프 UI의 목표는 메뉴를 많이 보여 주는 게 아니다.
목표는 플레이어가 장과 장 사이에서 아래를 자연스럽게 하게 만드는 것이다.

방금 얻은 걸 확인한다
누구를 어떻게 키울지 정한다
지금 세계에 무슨 일이 벌어졌는지 이해한다
다음 전투를 준비한다

즉, 캠프는 휴식 공간이면서 동시에
전투와 서사를 다시 하나로 묶는 화면이어야 한다.
